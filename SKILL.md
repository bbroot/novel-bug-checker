# 小说Bug检查专家

## 适用范围

**适用场景**：
- 检查小说中的逻辑漏洞和时间线矛盾
- 验证角色行为与设定的一致性
- 分析叙事节奏和信息密度
- 对小说章节或全文进行质量审查
- 检查伏笔埋设与回收情况

**不适用场景**：
- 语法检查或拼写纠正
- 写作灵感或创意建议
- 诗歌、剧本或非虚构作品
- 出版或营销咨询

**触发关键词**: 小说检查, bug检查, 逻辑漏洞, 角色一致性, 节奏分析, 叙事质量, 小说纠错

## 前置条件

```bash
pip install jieba
```

> ⚠️ 首次使用前请确认依赖已安装，否则脚本将无法运行。仅依赖 `jieba`（中文分词），无需重型NLP库。

## 核心能力

### 能力1：逻辑漏洞检测 — 时间线矛盾/因果断裂/设定冲突
识别情节中的逻辑断裂、时间线错误、信息知晓不合理和因果关系缺失。

### 能力2：角色一致性检查 — 性格突变/动机矛盾/对话风格偏移
确保角色行为符合其设定和发展轨迹，检测性格突变、动机矛盾和成长弧线断裂。

### 能力3：节奏结构分析 — 信息密度/高潮强度/伏笔管理
评估叙事节奏和信息密度，识别拖沓段落和节奏突变，检测伏笔回收情况。

## 命令列表

| 命令 | 说明 | 用法 |
|------|------|------|
| `logic` | 逻辑漏洞分析 | `python scripts/logic-analyzer.py <文件路径> [-o <输出文件>]` |
| `rhythm` | 节奏分析 | `python scripts/rhythm-analyzer.py <文件路径> [-g <类型>]` |
| `consistency` | 角色一致性检查 | `python scripts/consistency-checker.py <文件路径> [-o <输出文件>]` |

> 💡 脚本路径相对于技能根目录 `skills/novel-bug-checker/scripts/`

## Bug严重程度分级

| 等级 | 说明 | 示例 |
|------|------|------|
| 🔴 致命 | 破坏故事可信度的核心矛盾 | 主角能力突变无解释、时间线严重矛盾 |
| 🟠 严重 | 影响重要情节合理性的问题 | 角色动机矛盾、关键设定冲突 |
| 🟡 中等 | 影响阅读体验但可容忍 | 对话风格偶尔偏移、过渡生硬 |
| 🟢 轻微 | 细节瑕疵，不影响整体 | 常识性小错误、生活细节不合理 |

## 处理步骤

### Step 1：读取文本

**目标**：获取用户提供的小说文本内容

**执行**：
- 如果用户提供文件路径，使用 `read` 工具读取
- 如果用户直接粘贴文本，直接使用
- 如果用户提供章节范围，确认范围后读取对应部分

### Step 2：逻辑漏洞分析

**目标**：检测情节逻辑链中的断裂和矛盾

**执行**：
```bash
python scripts/logic-analyzer.py <文件路径> [-o <输出文件>]
```

**AI额外检查**（脚本之外必须执行）：
- 检查时间线：事件发生顺序是否合理，时间计算是否正确
- 检查因果链：每个结果是否有充分的原因支撑
- 检查信息知晓：角色是否知道不该知道的信息
- 检查能力边界：角色行为是否超出已建立的设定

### Step 3：角色一致性检查

**目标**：验证角色行为符合其设定和发展轨迹

**执行**：
```bash
python scripts/consistency-checker.py <文件路径> [-o <输出文件>]
```

**AI额外检查**（脚本之外必须执行）：
- 建立角色状态快照：记录每个关键节点角色的性格、能力、关系状态
- 对比相邻快照：变化是否有合理过渡
- 检查对话风格：语言是否符合角色背景和教育水平
- 验证成长弧线：角色发展是否有必要的经历和反思支撑

### Step 4：节奏结构分析

**目标**：评估叙事节奏和信息密度的合理性

**执行**：
```bash
python scripts/rhythm-analyzer.py <文件路径> [-g <类型>] [-o <输出文件>]
```

> 类型可选：玄幻（默认）/科幻/悬疑/爱情/武侠

**AI额外检查**（脚本之外必须执行）：
- 评估信息密度分布：是否有信息过载或过于稀疏的段落
- 检查高潮铺垫：高潮前是否有足够的紧张感积累
- 评估场景过渡：切换是否自然，有无突兀感
- 伏笔追踪：建立伏笔追踪表，标注已埋设和已回收的伏笔

### Step 5：生成报告

**执行**：按以下格式输出报告：

```
## 小说Bug检查报告

**检查范围**：[章节范围或完整作品]
**检查时间**：[时间戳]

### 问题统计
- 🔴 致命：X个
- 🟠 严重：Y个
- 🟡 中等：Z个
- 🟢 轻微：W个

### 问题详情

#### Bug #1
- **位置**：第X章 [段落引用]
- **严重程度**：[致命/严重/中等/轻微]
- **问题描述**：[详细说明]
- **根本原因**：[分析原因]

**修复建议**：
- 方案1：[描述] (难度: 低, 影响: [描述])
- 方案2：[描述] (难度: 中, 影响: [描述])
- 方案3：[描述] (难度: 高, 影响: [描述])
- ✅ 推荐方案：[方案N] — [推荐理由]

### 修复优先级
- ⚡ 立即修复：[问题列表]
- 📋 近期修复：[问题列表]
- 📌 可延迟修复：[问题列表]
```

### Step 6：验证修复（可选）

**目标**：如果用户提供了修改后的版本，重新运行检查验证修复效果

**执行**：重复 Step 2-5，对比前后报告，标注已修复和新增问题。

## 修复策略库

### 逻辑漏洞修复
| 策略 | 说明 | 适用场景 | 难度 |
|------|------|---------|------|
| 前向补充 | 在早期章节添加铺垫和解释 | 缺少前提条件 | 低 |
| 后向解释 | 通过回忆/对话补充说明 | 遗漏可事后圆回 | 中 |
| 规则调整 | 微调世界观设定使矛盾合理化 | 设定本身有缺陷 | 高 |
| 情节重构 | 重新设计相关情节 | 矛盾无法局部修复 | 高 |

### 角色问题修复
| 策略 | 说明 | 适用场景 | 难度 |
|------|------|---------|------|
| 渐进过渡 | 添加过渡场景展示变化过程 | 性格突变 | 低 |
| 动机强化 | 增加内心独白明确动机 | 行为不合理 | 中 |
| 关系发展场景 | 补充互动建立情感基础 | 关系发展不合理 | 中 |
| 对话调整 | 修改对话使其符合角色背景 | 语言风格偏移 | 低 |

### 节奏问题修复
| 策略 | 说明 | 适用场景 | 难度 |
|------|------|---------|------|
| 信息重分配 | 调整各章节信息密度 | 信息过载或稀疏 | 中 |
| 高潮增强 | 增加冲突层级和情感投入 | 高潮力度不足 | 中 |
| 过渡优化 | 添加场景/心理过渡段落 | 切换突兀 | 低 |
| 伏笔管理 | 建立伏笔追踪表确保回收 | 伏笔遗忘 | 低 |

## v2 增强：基于分布式状态的反向校验

> 当被检查的作品由 **novel-master v2 架构**创作（存在 `~/.qclaw/workspace/novels/<书名>/tracker/ncg.json`
> 与 `settings/characters/*/state.json`）时，bug-checker 可直接消费这些产物，做**更高精度的定向排查**：

1. **OOC 检测**：拿正文里的角色言行，对照其 `state.json` 的 `cognitive_bias_pack` 与 `relationships`，
   发现「角色突然说出了不属于自己偏差包的话 / 与关系网矛盾的行为」→ 直接定位 OOC 行。
2. **地理/时间穿越**：对照 NCG 节点的 `location` 与章节号，标出无过渡的位移（与 auditor 门禁互补，做二次把关）。
3. **伏笔断裂**：对照 `ncg.json` 的 `伏笔`/`因果` 标签节点与 `foreshadowing.json`，找出「埋了没收」或「收得不在窗口」。
4. **版本回退告警**：调用 `auditor.py check-version <书名>`，把角色状态版本异常列入 🔴 致命级。

**v2 联动命令**（需 novel-master 脚本存在）：
```bash
python ~/.qclaw/skills/novel-master/scripts/auditor.py check-version <书名>
python ~/.qclaw/skills/novel-master/scripts/ncg.py view <书名> causal
```

> 若作品并非 v2 架构（无 ncg.json），则回落到 v1 的 `logic/consistency/rhythm` 三脚本分析，不受影响。

## 约束

- **不要直接修改原文**：仅提供建议，不直接改写用户的小说内容
- **不要过度批评**：以建设性语气指出问题，避免打击作者信心
- **必须考虑上下文**：分析必须基于小说整体结构，不能断章取义
- **尊重类型差异**：不同类型小说（玄幻/都市/科幻）的合理性标准不同
- **每个Bug至少3种修复方案**：并标注推荐方案和理由
